# 고음팩토리 랜딩 리디자인 지시서 (Claude Code용)

목표: **지금의 “카드/박스 중심” 단조로운 구성**을, 투깝스 챌린지 페이지처럼 **세련된 편집형(에디토리얼) 랜딩**으로 바꾼다.  
핵심: 콘텐츠는 유지하되 **읽는 흐름(Story Flow)** + **타이포 중심** + **섬세한 모션**으로 “고급스럽게” 만든다.

> ⚠️ 제약
- **그라데이션 금지** (배경/버튼/텍스트 모두)
- 현 콘텐츠(문구/영상/가격/연락처)는 최대한 유지
- 전체 톤은 이미 쓰는 `bg-[#040814]` 기반의 **다크 & 미니멀**
- “인위적인 박스 안 문장” 느낌을 줄이고, **페이지를 따라 자연스럽게 읽히는 문단**으로 재배치

---

## 1) 리디자인 방향 (투깝스 챌린지 무드 참고)
정확한 코드/DOM을 가져오는 게 아니라, “느낌”을 가져온다:

### Visual 키워드
- **큰 타이포 + 넓은 여백**으로 고급스럽게
- 섹션 간 **리듬감 있는 구분**: (얇은 라인, 섬세한 배경 톤 변화, 스티키 라벨)
- 카드 남발 금지 → **필요한 정보만 ‘모듈’로 강조**
- “증거(Proof)” 파트는 **영상 + 한두 문장 훅 + 짧은 근거**로 빠르게 납득

### UI 키워드
- Sticky Top Nav는 유지하되, 본문은 “스토리텔링” 위주
- 버튼은 **단색 + 미세한 호버/프레스** (그라데이션 X)
- 배경 장식은 가능: **노이즈/그리드/희미한 라이트 스팟(단색, low opacity)** 정도만

---

## 2) 정보 구조 재설계 (Story Flow)
현재 구조는 섹션마다 박스가 많아서 “끊김”이 큼.  
아래처럼 “한 번에 읽히는 흐름”으로 재배치.

### A. Hero: 한 문장씩 내려오며 설득
- H1 (큰 타이포)
- 2~3줄 서브카피 (문단형, 박스 X)
- 영상(소개) 1개
- 프로필은 “카드” 말고 **얇은 라인/약한 배경으로만** (또는 한 줄 소개 + 작은 아바타)

### B. Proof: BEFORE/AFTER는 ‘증거 스트립’처럼
- 섹션 타이틀 아래에 “짧고 강한 주장”
- `FeatureCard` 남발 대신:
    - (1) **Claim 문장(큰 글씨)**
    - (2) **근거 2~3줄**
    - (3) 영상 1개
    - 이런 패턴을 반복

### C. 1억 케이스: 케이스 스터디처럼 구성
- “상황 → 난이도 → 결과” 3단 구성으로 짧게
- 영상 1개 + 아래는 문단형 설명 + 한 줄 인용(Quote)

### D. 보장/환불: ‘리스크 리버설’ 섹션
- “전액환불”을 큰 타이포로, 상세는 문단형
- “80대 20분 이해”는 **뱃지/하이라이트**로 옆에 붙이기

### E. Curriculum: 지금 박스 UI는 좋지만, 상단에 “요약/약속”을 더
- 17강 구성 → “이 강의가 해결하는 것” 3줄 요약 추가
- 리스트는 유지하되, 전체 박스의 존재감을 줄이고 **섹션 배경 흐름 안에 자연스럽게** 배치

### F. Pricing & Contact: 가격은 “한 방”으로
- 정가/할인가/프로모션가를 “타이포 레이아웃”으로
- 연락처(Contact)는 별도 박스로 감싸지 말고 깔끔하게

---

## 3) 스타일 가이드 (Tailwind 기준)

### 색/배경
- 기본 배경: `bg-[#040814]`
- 섹션 배경 변주(그라데이션X):
    - `bg-white/[0.02]`, `bg-white/[0.03]` 같은 **미세한 톤 차이**
    - 섹션 상단에 `border-t border-white/10` 같은 라인으로 구분

### 타이포
- Hero H1: `text-4xl md:text-6xl font-black tracking-tight`
- 섹션 타이틀: `text-2xl md:text-4xl font-semibold`
- 본문: `text-white/75 leading-8`
- 강조는 `Emphasis` 유지하되 남발하지 말고 “포인트 문장”에만

### 레이아웃/여백
- max-width는 유지: `max-w-3xl`
- 섹션 패딩 늘리기: `py-14 md:py-20`
- 문단형 레이아웃: 카드 대신 `space-y-6`로 흐름 만들기

### 컴포넌트 톤
- 카드가 필요할 때만 사용: “주의사항”, “핵심 요약”, “연락처” 정도
- 나머지는 “문단 + 얇은 라인 + 작은 뱃지”로 해결

---

## 4) 모션/애니메이션 (과하지 않게, 하지만 ‘살아있게’)

### 라이브러리
- `framer-motion` 사용 권장 (Next.js App Router 환경)
- IntersectionObserver 기반 `Reveal` 컴포넌트 만들어서 공통 적용

### 규칙
- 기본 Reveal: 아래에서 위로 10~16px + fade-in
- duration 0.45~0.6s / easeOut
- 섹션 내 요소는 `stagger`로 0.06~0.1s 정도만
- hover 모션은 미세하게: border/opacity 변화 + translateY 1~2px 정도

### 적용 위치
- SectionTitle
- Claim 문장(큰 문장)
- 영상(YouTubeEmbed)
- 커리큘럼 리스트 아이템
- 가격 영역

---

## 5) 새로 만들 “프리미티브” 컴포넌트 제안
> 기존 컴포넌트를 무조건 다 갈아엎지 말고, 아래 프리미티브를 추가해서 리팩토링.

1) `Reveal.tsx`
- children을 감싸서 스크롤 진입 시 애니메이션
- 옵션: `delay`, `y`, `once`

2) `StoryBlock.tsx`
- 박스 없이 문단 흐름을 만드는 블록
- 구성: `eyebrow(작은 라벨) + headline(큰 문장) + body(문단)`

3) `ClaimRow.tsx`
- Proof 섹션에서 반복되는 패턴
- 왼쪽에 작은 인덱스(01/02/03), 오른쪽에 claim+근거

4) `Quote.tsx`
- 한 줄 강한 인용/결론 문장
- 얇은 라인 + italic(또는 font-medium) 정도

5) `SectionShell.tsx`
- 섹션 공통 wrapper
- `id`, `tone(0/1/2)` 받아서 배경톤만 살짝 다르게

## 6) 구체 작업 지시 (Claude Code TODO)
아래를 그대로 실행해줘.

### 6-1. 페이지 구조 리팩토링
- `HomePage`의 각 section을 `SectionShell`로 감싸고
- 박스형 설명 카드(`rounded-2xl border bg-white/5 p-6`)는 절반 이상 제거
- 설명 텍스트는 `StoryBlock`/문단형 레이아웃으로 옮겨서 “자연스럽게 이어지게” 구성

### 6-2. Proof 섹션을 “ClaimRow + 영상” 패턴으로 재작성
- FeatureCard 2개는 제거/축소하고 ClaimRow로 바꿀 것
- 흐름 예시:
    - ClaimRow(“2옥 솔# 고음불가였다” + 근거)
    - YouTube(beforeAfter1)
    - ClaimRow(“수강생 사례로 검증됨” + 근거)
    - YouTube(students)
    - Quote(“단연 대한민국 최고의 고음강의일 겁니다.”)

### 6-3. 1억 케이스 섹션 문단화
- 현재 큰 박스 설명 → `StoryBlock` 2~3개 문단으로 분할
- 중간중간 Emphasis는 딱 필요한 곳(1억원/4개월)만

### 6-4. 보장 섹션 “리스크 리버설” 타이포 강화
- “변화가 없다면 전액환불” 문장을 섹션 내부에서 한 번 더 큰 타이포로 강조
- “80대 20분”은 작은 뱃지로 옆에 붙이거나, Claim 형태로 분리

### 6-5. CurriculumBox 존재감 조정
- CurriculumBox 외곽 박스 톤을 더 약하게:
    - border opacity 줄이고, 배경 `bg-white/[0.03]` 정도
- 상단 요약 바에 “이 강의가 해결하는 것(3가지)” 한 줄 추가

### 6-6. Pricing 영역 재디자인 (가장 중요)
- border/box 없이 타이포로만 세련되게
- 예시 레이아웃:
    - (작게) 정가 49만원 (strike)
    - (중간) 할인가 29만원 (strike)
    - (크게) 프로모션 25만원
    - (작게) “카톡 문의 후 결제 안내” 같은 한 줄
- CTA 버튼 1개 추가 추천:
    - “카카오톡으로 문의하기” (Contact의 카카오 링크 재사용)

### 6-7. 모션 적용
- SectionTitle / StoryBlock / 영상 / 커리큘럼 아이템 / 가격 라인에 `Reveal` 적용
- 스크롤에 따라 “부드럽게 등장”하는 느낌 만들기

## 7) 산출물 요구사항
- 코드 수정 결과는 “바로 붙여넣으면 빌드되는 수준”으로 제공
- 새 컴포넌트 파일 경로는 프로젝트 컨벤션에 맞게:
    - `src/component/Reveal.tsx`
    - `src/component/SectionShell.tsx`
    - `src/component/StoryBlock.tsx`
    - 등으로 생성
- 기존 컴포넌트(`SectionTitle`, `Contact`, `YouTubeEmbed`, `Emphasis`)는 최대한 재사용
- 디자인 결과물은 “카드 남발”이 아니라 “문단 흐름”이 중심이어야 함

## 8) 최종 체크리스트 (완료 조건)
- [ ] 박스/카드 개수가 확실히 줄었고, 문장이 페이지를 따라 자연스럽게 이어진다
- [ ] 섹션 간 리듬(여백/라인/톤)이 생겼다
- [ ] 애니메이션은 과하지 않고 고급스럽게 느껴진다
- [ ] 그라데이션이 없다
- [ ] 가격 파트가 가장 “고급스럽고 설득력 있는 타이포”로 보인다
